# /the-one-move

Run BEFORE writing UI for a new brand/marketing surface. It decides the **structure** the AI default would never commit to, then hands the approved skeleton to `/impeccable` to build.

This is the generative, upfront complement to `/improve-platform` and `/ui-check` (which diagnose slop after the fact). It is deliberately narrow: it owns **layout, composition, hierarchy, subject, the one move** — not palette, fonts, motion, or copy. Those belong to `impeccable`.

## Flow

1. **Load the skill.** Follow `skills/the-one-move/SKILL.md` exactly — the setup, the seven structural tells, the one-move rule, the anti-monoculture guard, and the output format.
2. **Establish intent.** If who-it's-for and the one-thing-they-must-do are unclear, ask 2–3 sharp questions first.
3. **Produce the structural brief** (the 7-part output). One screen, no styling decisions.
4. **Get approval.** Iterate the structure with the user until they commit. Do not start building.
5. **Hand off.** On approval, invoke `/impeccable shape` (or `craft`) with the brief as the structural spec. Impeccable owns execution and the slop bans from there.

## Output

The one-page structural brief: intent · subject · the one move · composition skeleton (as plain wireframe boxes, with the desktop→mobile change) · hierarchy · tells broken · the risk + mitigation. Then the explicit hand-off to `impeccable`.
